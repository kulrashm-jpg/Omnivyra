# Campaign Launch From Opportunity

Allows users to create a campaign directly from a **campaign opportunity** generated by the intelligence system. Flow: **Campaign Opportunity → Campaign Blueprint → Campaign Builder**. Does not modify existing intelligence pipeline components.

---

## 1. API Endpoint

**POST** `pages/api/campaigns/create-from-opportunity.ts`

### Input (body)

```json
{
  "opportunity_id": "uuid-of-campaign-opportunity",
  "company_id": "uuid-of-company"
}
```

### Response (201)

```json
{
  "campaign_id": "uuid",
  "campaign_name": "Create blog posts explaining how AI Productivity Automation improves productivity.",
  "opportunity_id": "uuid",
  "blueprint": {
    "campaign_id": "uuid",
    "duration_weeks": 4,
    "content_pillars": ["Educational Content", "Practical Use Cases", "Expert Insights", "Customer Stories"],
    "weeks": [
      {
        "week_number": 1,
        "phase_label": "Educational Content",
        "primary_objective": "Educational Content",
        "topics_to_cover": [],
        "platform_allocation": {},
        "content_type_mix": ["post"],
        "cta_type": "None",
        "weekly_kpi_focus": "Reach growth"
      },
      "..."
    ]
  }
}
```

### Errors

- **400** — Missing `opportunity_id` or `company_id`
- **403** — Access denied to company
- **404** — Campaign opportunity not found
- **500** — Server error (e.g. DB failure)

---

## 2. Data Source

Opportunity is loaded from **`campaign_opportunities`** with:

- `opportunity_title`
- `opportunity_description`
- `opportunity_type`
- `momentum_score`
- `keywords`
- `cluster_id`
- `theme_id`

---

## 3. Campaign Generation

- **campaign_name** = `opportunity_title`
- **campaign_description** = `opportunity_description`
- **campaign_type** (in planning_context) = `"trend_campaign"`
- Campaign record: `status: 'draft'`, `current_stage: 'planning'`

---

## 4. Content Pillars (by opportunity_type)

| opportunity_type     | Pillars |
|----------------------|--------|
| content_marketing    | Educational Content, Practical Use Cases, Expert Insights, Customer Stories |
| thought_leadership   | Executive Perspective, Industry Trends, Strategic Implications, Future Outlook |
| product_positioning  | Product Benefits, Customer Outcomes, Implementation Strategy, Competitive Advantage |
| industry_education   | Market Trends, Technology Overview, Industry Adoption, Best Practices |

Blueprint **weeks** are built as one week per pillar (phase_label = pillar name).

---

## 5. Service

**File:** `backend/services/opportunityCampaignGenerator.ts`

**Main function:** `generateCampaignFromOpportunity(opportunityId, companyId, userId)`

**Steps:**

1. Load opportunity from `campaign_opportunities` (throw if not found).
2. Build campaign blueprint (content pillars + one week per pillar).
3. Create campaign record (`campaigns` table).
4. Create campaign version (`campaign_versions`) with metadata and planning_context.
5. Log `campaign_created_from_opportunity`.
6. Return `{ campaign_id, campaign_name, opportunity_id, blueprint }`.

---

## 6. Campaign Metadata (campaign_snapshot)

Stored in `campaign_versions.campaign_snapshot.metadata` and planning_context:

```json
{
  "source": "trend_opportunity",
  "opportunity_id": "...",
  "theme_id": "...",
  "cluster_id": "...",
  "momentum_score": 0.82
}
```

`planning_context` also includes: `campaign_type`, `content_pillars`, `keywords`, and the same ids for downstream use.
